Fix `cmd/update.rs` so that `--mode plain` or `--mode json` without `--yes` or `--dry-run` fails before fetching GitHub/cache data.

Current behavior appears to fetch tags and plan candidates first, then render.
SPEC requires failing early when possible.

Acceptance criteria:
- `actioneer update --mode plain .github` fails with a clear message unless `--yes` or `--dry-run` is also given.
- `actioneer update --mode json .github` fails the same way.
- `actioneer update --mode tui .github` still proceeds to TUI selection.
- `--dry-run` still works in plain/json mode.
- `--yes` still works in plain/json mode.
- The error is produced before network/cache work begins.
Keep tests green.
