Implement SHA/comment mismatch detection in audit.

Behavior:
- When a ref looks like a SHA and has a version comment, verify the comment tag matches the SHA.
- If the SHA comment cannot be verified because GitHub/cache data is missing in offline mode, fail clearly.
- If verification fails, report a `sha_comment_mismatch` finding with `expected_sha` set.
- If verified, allow the ref as secure.

This requires audit to fetch/use GitHub/cache data when available and fail predictably when offline data is missing.
