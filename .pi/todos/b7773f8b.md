Implement the simplified TUI for update selection.

Requirements:
- Uses `ratatui` + `crossterm`.
- Only update uses TUI; audit and `audit --fix` never do.
- Receives `&[Candidate]` and returns selected indexes.
- Does not copy candidates into a new domain model.
- Local state only: cursor position, selected bools.
- Default selection: all candidates selected.
- Minimal keybindings:
  - space: toggle candidate
  - a: select all
  - n: select none
  - enter: confirm
  - q/esc: cancel
- Shows per row: action repo, file:line, current ref -> target ref, target version, reason/note.
- If TUI is requested but stdin/stdout is not a TTY, fail with a clear message.

After this, update `--mode tui` without `--yes` or `--dry-run` should open selection, then patch selected candidates.
Keep tests green.
