Extend rule condition parsing to support the language defined in SPEC.md:

- `==` and `!=` (already supported)
- `starts_with(value, prefix)`
- `&&`
- `||`

Metadata fields should include:
- `ActionRepoOwner`
- `ActionRepoName`
- `ActionRepo`
- `ActionPath`
- `WorkflowFile`
- `CurrentRef`
- `CurrentRefKind`

Do not build a full expression evaluator or scripting language. Keep the parser small and fail clearly on unsupported conditions.
Add tests for each operator and metadata field.
Keep tests green.
